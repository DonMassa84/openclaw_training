# Mini AI Governance

## Goal

Define minimum rules for safe, traceable and portfolio-ready AI use in local IT operations workflows.

## Principles

1. Local-first where possible.
2. No publication of sensitive raw data.
3. Human review before external output.
4. Logging without private content.
5. Documentation before scaling.
6. Security check before commit.
7. Automation only with fallback.

## Human-in-the-Loop

Human approval is mandatory for:

- GitHub publication
- LinkedIn posts
- recruiter-facing portfolio artifacts
- security documents
- workflows using sensitive cases
- system-changing automation
